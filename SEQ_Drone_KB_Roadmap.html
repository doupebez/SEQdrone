1. Platform Overview

The SEQ Drone Inspections AI Platform is a web-based application that enables drone pilots and operators to upload inspection imagery, run AI-powered defect analysis, and generate professional PDF inspection reports — all from a single dashboard.

2. User Authentication

Users access the platform via email + password authentication. New accounts are created through the Sign Up page, and returning users log in via the Login page.

3. Core Workflows

3.1 Standard Inspection Survey (Drone + AI)

This is the primary workflow for roof, solar, facade, bridge, tower, turbine, and tank inspections.

Upload Images → Job Context → AI Analysis → Review & Refine → Generate PDF
1

Upload Images

  • Drag & drop or click to upload high-resolution drone images (JPG/PNG)
  • Multiple images supported (up to 50 per survey)
  • System automatically extracts EXIF metadata: GPS coordinates, altitude, pitch, heading, timestamp, and drone model
  • "Set as Overview" — select any image to feature on the report cover page
2

Job Context

  • Select Asset Type from 8 categories: Roof, Solar Panels, Building Facade, Bridge, Comm Tower, Wind Turbine, Water Tank, Height Safety
  • Enter job title/reference, client name, and asset location
  • Optional AI Prompt — free-text instructions for the AI (e.g., "Look for cracked tiles and hail damage on the north side")
  • Existing Asset Detection — if the asset has been surveyed before, quick-select buttons auto-populate details for chronological tracking
3

AI Analysis

Each image is processed individually through Google Gemini 2.0 Flash. For each image the AI:

  • Describes the visible section of the asset
  • Identifies all defects, damage, and maintenance issues
  • Classifies each by type, severity, and component
  • Pins the exact defect location using focal-point coordinates
  • Provides professional descriptions and actionable recommendations with timeframes

If Training Rules exist, they are automatically injected into the AI's system prompt for improved accuracy.

4

Review & Refine

Users can review, edit, and clean up findings before generating the report:

  • Browse findings — each card shows title, severity, confidence score, description, recommendation, and source image
  • Delete false positives — hover to reveal the trash icon, click and confirm to remove
  • Visual/Thermal toggle — simulated thermal vision mode
  • Image Markup — draw safe-access or exclusion-zone paths, add text notes, record thermal anomaly temperatures
  • Measurement Tools — calibrate a scale reference, then draw measurement lines with auto-calculated real-world distances
  • Geospatial Site Map — if GPS data exists, a tactical map shows flight path, camera positions, and finding locations
5

Generate Report

Professional A4 PDF report includes:

  • Branded cover page with health score and client details
  • Executive summary with severity breakdown chart
  • Full-page site overview image
  • Image gallery with finding counts per image
  • Detailed finding pages with annotated images, zoomed crops, metadata, and colour-coded recommendations
  • Survey images appendix
  • Branded header/footer on every page with page numbers

3.2 Height Safety Inspection (Manual Entry)

A dedicated workflow for height safety equipment audits — anchor points, static lines, davit systems. This workflow bypasses image upload and AI analysis entirely.

Key Difference

Height Safety inspections are fully manual-entry. The operator records inspector details, equipment registers, and defects directly in the form without AI involvement.

Inspector & Site Tab

  • Inspector name, licence/cert number, company
  • Building description
  • Inspection date and next-due date (auto-populated from job context)

Equipment Register Tab

  • Equipment grouped by Location (e.g., "Roof", "Level 25", "Carpark P1")
  • Per-item: ID, Equipment Type, Fixing Method, kN Rating, Test Method, Rope Access, Fall Arrest, Result (Pass/Fail/No Access)
  • Bulk duplicate — instantly create multiple copies from a template row
  • Summary stats bar: Total / Pass / Fail / No Access

Defects Tab

  • Equipment IDs affected, location, risk rating (Low → Extreme)
  • Description, photo upload, safe-for-temporary-use flag
  • Resolution / recommendation
  • Conclusion & recommendations text area

Height Safety PDF Report Includes:

  • Branded cover page with pass/fail/no-access summary
  • Australian Standards references (AS 1891, AS 5532, ISO 22846)
  • Certification requirements, OHS legislation, risk assessment methodology
  • Equipment registers by location (paginated at 27 rows per page)
  • Defect detail pages with photos, risk ratings, and resolutions

Bar chart of top 3 defect types by percentage across the portfolio.

3.4 AI Training Console

The Training Console (/training) creates a feedback loop that makes the AI smarter with every inspection.

Review Past Analyses → Submit Feedback → Generate Rules → AI Improves

Feedback Categories

  • False Positive — AI detected something that isn't a defect
  • Missed Defect — AI missed a visible issue
  • Wrong Severity — severity rating was incorrect
  • Wrong Category — defect type was misclassified
  • Good Detection — positive reinforcement
  • General Note — any other observation

Clicking "Generate Rules from Feedback" sends all feedback to Gemini, which distills it into concise, actionable rules that are automatically injected into every future analysis.

Operators can also add manual rules directly, and toggle rules on/off without deleting them.

4. Data Architecture

Database Tables (Supabase)

TablePurpose
surveysAll survey records — job data, analysis results, images, health scores
training_feedbackOperator corrections and feedback entries for past analyses
training_rulesDistilled AI training rules with active/inactive toggle

Key Data Objects

Survey Record

id, date, status (Draft/Finalized), jobData (title, client, location, type, assetId), analysisResult (summary, damages[], healthScore), thumbnail, findingCount

Damage / Finding

id, title, severity, defectType, conditionRating, description, recommendation, component, areaLocation, estimatedSize, focalPoint, confidenceScore, imageIndex, annotations

Height Safety Inspection

Inspector details, building info, dates, equipment[] (type, fixing, kN, test, result), defects[] (risk, description, photo, resolution), conclusion

Training Rule

id, rule text, source (which feedback it came from), active toggle, createdAt timestamp

Part 2

Development Roadmap

Strategic feature roadmap organised by phase, effort, and business impact.

Phase 1
Quick Wins
Estimated: 1–2 weeks

1.1 Client Report Portal

Goal: Allow clients to view and download reports without operator involvement.

  • Read-only dashboard filtered by client
  • Shareable link per survey (password-protected or magic link)
  • Download PDF directly from the portal
  • No login required for link-based access

1.2 Email Report Delivery

Goal: Send finalised reports directly to the client's email.

  • "Email to Client" button on the review page
  • Auto-attach PDF as email attachment
  • Customisable email template with company branding
  • Delivery confirmation logging

1.3 Chronological Comparison Reports

Goal: Show side-by-side progression for repeat inspections.

  • Auto-detect same-asset surveys via assetId (already in data model)
  • Health score trend chart over time
  • New / Resolved / Worsened finding indicators
  • Exportable comparison PDF

1.4 Bulk Image Compression

Goal: Speed up uploads and reduce processing time.

  • Client-side resize before upload (4000px max width)
  • Compress to 80% quality JPEG, preserving EXIF
  • Show file size savings to user
Phase 2
Core Enhancements
Estimated: 3–4 weeks

2.1 Real Thermal Imaging Support

Goal: Process actual thermal (IR) drone imagery alongside visual imagery.

  • Upload both RGB and thermal images per survey point
  • AI analyses dual-spectrum data in context
  • Temperature gradient detection for solar panel hotspots
  • Side-by-side thermal/visual view and thermal data in reports

2.2 Job Scheduling & Calendar

Goal: Manage upcoming inspections and schedule follow-ups.

  • Calendar view of past and upcoming inspections
  • Auto-schedule next-due dates from height safety reports
  • Email/push reminders for upcoming inspections
  • Recurring inspection templates

2.3 Report Templates & Customisation

Goal: Customise report branding and layout per client.

  • Upload company logo for PDF reports
  • Configurable colour scheme and cover page text
  • Optional sections (hide/show recommendations, executive summary)
  • Custom disclaimer / terms and conditions

2.4 Enhanced Mobile Experience

Goal: Make the survey workflow usable from a tablet on-site.

  • Responsive redesign for iPad/tablet
  • Camera integration for direct capture
  • Touch-friendly markup tools
  • Offline draft saving with sync
Phase 3
Scale & Automate
Estimated: 4–6 weeks

3.1 Multi-Organisation Tenancy

Goal: Support multiple companies/teams using the same platform.

  • Organisation accounts with team member management
  • Role-based access: Admin, Operator, Client, Reviewer
  • Per-organisation branding and settings
  • Separate data isolation per organisation

3.2 Native Mobile App (PWA or React Native)

  • Progressive Web App with offline support
  • Push notifications, GPS auto-tag, camera-first workflow
  • Quick height safety entry with barcode/NFC scanning

3.3 Batch Processing Pipeline

  • Background job queue for 100+ image surveys
  • Progress notifications and automatic retry on failure
  • Parallel processing for speed

3.4 Image CDN & Storage

  • Move images from base64-in-DB to Supabase Storage or Cloudflare R2
  • WebP conversion and automatic thumbnailing
  • Dramatically reduced database size
Phase 4
Intelligence & Analytics
Estimated: 6–8 weeks

4.1 Predictive Maintenance Engine

  • AI-powered deterioration forecasting across chronological inspections
  • Time-to-failure predictions and remaining useful life estimates
  • Auto-generated maintenance priority schedules
  • Cost estimation for recommended repairs

4.2 Portfolio Analytics Dashboard

  • Interactive charts: defect trends, health scores over time
  • Client and location-level aggregation
  • Exportable analytics reports (PDF or Excel)

4.3 Natural Language Search

  • "Show me all critical corrosion findings from the last 6 months"
  • Powered by Gemini with context from the survey database
Phase 5
Enterprise Features
Estimated: 8–12 weeks

5.1 API & Integration Layer

  • REST API for survey data, findings, and reports
  • Integration with asset management systems (Fiix, UpKeep) and CRMs
  • Webhook notifications on survey completion

5.2 Compliance & Audit Trail

  • Full audit log, version history, digital signatures on reports
  • GDPR/privacy compliance and data retention policies

5.3 White-Label Solution

  • Fully customisable branding (logos, colours, domain)
  • Per-tenant configuration and custom report templates
  • SaaS licensing or revenue sharing model

5.4 Video & Livestream Analysis

  • Upload video, auto-extract key frames for analysis
  • Real-time analysis during live drone flights (DJI SDK)
  • Highlight reel generation for client presentations
